# 前端 UI 重设计功能描述

本文档面向准备重新设计 Muse 前端 UI 的设计师、产品经理与前端工程师。它描述当前产品需要承载的功能、角色、流程、状态与约束，不要求沿用现有页面布局或视觉风格。技术结构见 [`FRONTEND.md`](./FRONTEND.md)，接口契约见 [`API.md`](./API.md)。

## 设计目标

Edge Muse 是一个内部开通制的多服务商图片生成平台。前端重设计应优先解决三件事：

1. 让普通用户更快完成「选案例或空白创作 → 优化 Prompt → 生成 → 查看与复用结果」。
2. 让管理员和系统管理员在高密度数据里高效完成开户、配额、密钥、审计与内容配置。
3. 保留现有安全、配额、服务商能力、异步任务与多语言约束，避免设计出后端不支持的交互。

本项目没有公开注册、忘记密码、邮件邀请或公开支付流程。账号、密钥与额度由管理员人工开通。

## 用户角色

| 角色                  | 核心目标                                | 主要功能                                                                          |
| --------------------- | --------------------------------------- | --------------------------------------------------------------------------------- |
| 普通用户 `user`       | 在配额内生成图片、查看历史、复用 Prompt | AI 图像生成、经典工作台、历史、大图查看、个人资料、改密码、公告                   |
| 管理员 `admin`        | 管理名下用户与配额                      | 用户列表、创建用户、加配额、启停用户、重置密码、查看用量                          |
| 系统管理员 `sysadmin` | 管理全局平台配置与运维审计              | 全局看板、服务商密钥、管理员/用户、生成入口开关、案例库、公告、会话审计、系统偏好 |

权限边界必须清晰可见。普通用户不应看到后台管理入口；管理员不应看到服务商密钥与全局系统设置；系统管理员可访问全部管理功能。

## 登录与账号

当前实现参考 [`web/src/views/auth/Login.vue`](../web/src/views/auth/Login.vue)。

### 登录页功能

- 输入登录标识和密码。登录标识可能是用户名或邮箱。
- 读取 `/api/config`，如果存在 Turnstile site key，则显示 Cloudflare Turnstile 验证。
- 登录成功后根据 `redirect` 参数返回原页面；无 `redirect` 时进入默认首页。
- 登录失败显示错误提示。
- 登录页支持语言切换。

### 账号设置

- 个人资料：用户可修改昵称，调用 `PATCH /api/me`。
- 安全设置：用户输入旧密码和新密码修改密码。
- 忘记密码不在用户自助范围内，由管理员手动重置。

## 普通用户创作功能

普通用户的核心体验由「AI 图像生成」和「工作台」两个入口组成。两者都复用同一套后端生成任务、配额、服务商能力和图片查看能力。

### 服务商能力约束

前端必须从 `/api/me` 的 `providerCapabilities` 读取当前账号能力：

- `supportedModes`：可用模式，包含 `text2image`、`image2image` 中的一部分。
- `supportedSizes`：可用画布尺寸。
- `maxReferenceImages`：参考图最大数量，可能为 `null`。
- `requestFormat`：服务商请求形态，例如 Cubence 使用 `openai_images`。

UI 必须隐藏或禁用不支持的模式。当前工作台只展示文生图与图生图。

### AI 图像生成页

当前实现参考 [`web/src/views/ai-image/AiImageGeneration.vue`](../web/src/views/ai-image/AiImageGeneration.vue)。

这个页面应被视为普通用户的轻量创作主入口，核心流程是：

1. 选择案例，或从空白创作开始。
2. 查看案例详情，包括示例图、标题、分类、摘要、标签、推荐尺寸与来源归因。
3. 套用案例 Prompt，或通过 Prompt 助手生成最终 Prompt。
4. 选择生成模式、尺寸，图生图时上传参考图。
5. 提交生成，查看实时进度。
6. 成功后查看结果图；失败时看到失败原因并可重试。

#### 案例选择

- 展示已发布案例列表，只显示用户可用模式支持的案例。
- 支持分类筛选。
- 支持推荐/精选标记。
- 案例卡片至少包含缩略图、标题、摘要、标签、推荐尺寸与来源信息。
- 桌面端应能同时浏览列表和详情；移动端可用底部抽屉或详情页。
- 用户可以放弃案例，直接进入空白创作。

#### Prompt 助手

当前实现参考 [`web/src/views/ai-image/PromptAssistantPanel.vue`](../web/src/views/ai-image/PromptAssistantPanel.vue)。

- Prompt 助手通过自然语言对话帮助用户形成最终 Prompt。
- 助手不会直接触发生图，最终仍需用户点击生成。
- 助手应感知当前案例、生成模式、服务商尺寸能力、参考图数量和参考图描述。
- 支持多轮对话，当前最多 8 轮。
- 支持「直接生成最终 Prompt」。
- 支持重开对话、复制最终 Prompt、回填到生成表单。
- 如果助手不可用，应隐藏入口或呈现明确降级状态。

#### 生成表单

当前实现参考 [`web/src/views/ai-image/AiImagePromptPanel.vue`](../web/src/views/ai-image/AiImagePromptPanel.vue)。

- 模式选择：文生图、图生图，按服务商能力过滤。
- 尺寸选择：按服务商支持尺寸展示；案例推荐尺寸不可用时显示替代说明。
- Prompt 编辑：支持手动编辑、清空、复制、从案例重置。
- 图生图参考图：支持拖拽、点击选择、粘贴图片，支持移除预览。
- 参考图说明：给 Prompt 助手使用，帮助描述参考图。
- 提交按钮：无 Prompt、能力不支持、任务运行中、参考图数量非法时禁用。

#### 任务与结果

- 提交后进入任务运行态，显示状态文案、进度条、Prompt 摘要。
- 结果区展示生成图片网格；单图应尽可能大且可完整查看。
- 点击结果图进入全屏图片查看器。
- 失败时展示业务可读标题、失败消息和重试按钮。
- 任务运行期间应避免用户误改会影响当前任务的关键参数。

### 工作台

当前实现参考 [`web/src/views/workspace/Workspace.vue`](../web/src/views/workspace/Workspace.vue)。

工作台是经典生成入口，适合继续历史会话或更工程化的参数操作。

#### 会话与模式

- 支持新建生成会话。
- 支持通过 `/workspace/s/:sessionId` 打开指定会话。
- 支持编辑新会话标题，历史会话标题只读或按后端能力控制。
- 模式包括文生图、图生图，按当前服务商能力过滤。
- 模式切换在任务运行中应禁用。

#### 一次性任务模式

- 左侧或主输入区：Prompt、尺寸、张数、参考图。
- 系统管理员可输入自定义张数；普通用户通常使用固定选项。
- 图生图支持上传参考图，参考图数量受服务商限制。
- 提交后右侧结果区显示进度、失败或图片结果。
- 已有结果应支持缩略图切换和大图打开。

### 历史

当前实现参考 [`web/src/views/history/History.vue`](../web/src/views/history/History.vue)。

#### 历史列表

- 会话卡片网格，展示封面图、标题、最近更新时间、状态、图片数量。
- 支持搜索标题或 Prompt。
- 支持排序：最近、最早、任务最多。
- 支持分页和页码跳转。
- 空状态、加载状态要明确。

#### 历史详情

- 展示该会话的生成结果，支持上一条/下一条结果切换。
- 图片结果区支持多图网格，点击进入大图查看器。
- 侧边信息区展示创建时间、任务状态、生成进度、总图数、成功图数、失败图数、模式、尺寸、参考图数量、模型。
- 展示参考图缩略图。
- 展示 Prompt，长 Prompt 需要可滚动阅读。
- 失败任务展示错误详情。

### 图片查看器

当前实现参考 [`web/src/components/image/ImageViewer.vue`](../web/src/components/image/ImageViewer.vue)。

全屏图片查看器应在 AI 图像生成、工作台、历史和会话审计中复用：

- 显示图片标题、尺寸、文件大小、关联任务。
- 支持上一张/下一张。
- 支持键盘 Esc 关闭、左右箭头切换。
- 支持缩放、重置缩放、滚轮缩放、放大后拖拽平移。
- 支持复制 Prompt。
- 支持下载图片。
- 在允许删除的上下文中支持删除消息图片；只读场景隐藏或禁用删除。
- 底部展示 Prompt 摘要，移动端可收起。

## 管理员功能

当前实现参考 [`web/src/views/admin/UserList.vue`](../web/src/views/admin/UserList.vue)。

### 用户列表

- 表格展示序号、用户、角色、额度、最后登录、最后生成、生成次数、状态、操作。
- 支持搜索登录名/邮箱。
- 支持按启用/禁用状态筛选。
- 系统管理员视角支持按角色筛选。
- 支持分页、上一页/下一页、页码跳转。
- 行点击或按钮打开用户详情侧栏。

### 用户创建与编辑

- 管理员可创建普通用户。
- 系统管理员可选择创建普通用户或管理员。
- 创建字段：登录用户名、显示昵称、密码、可选邮箱、服务商密钥、初始额度。
- 系统管理员可编辑用户昵称、状态、服务商密钥、总额度和可选新密码。
- 密码要求至少 8 位，具体强规则以后端返回为准。

### 配额与状态

- 支持给用户追加额度。
- 加额度时展示操作者自身剩余额度；系统管理员可能为无限额度。
- 支持启用/禁用用户。
- 支持重置用户密码。
- 用户详情侧栏展示额度使用率、剩余额度、任务总数、状态分布、模式分布、30 天趋势和配额流水。

## 状态与反馈规范

重设计必须覆盖以下状态，而不是只画成功态：

- 加载中：列表、详情、表单提交、密钥测试、任务运行、Prompt 助手请求。
- 空状态：无历史、无案例、无公告、无密钥、无用户、无结果图。
- 错误状态：登录失败、表单校验失败、接口失败、Prompt 助手失败、服务商生成失败。
- 禁用状态：任务运行中、额度不足、模式不支持、参考图数量超限、无权限、保存中。
- 成功反馈：保存成功、发布成功、复制成功、密码修改成功、配额追加成功。
- 危险操作确认：删除密钥、删除公告、归档内容、禁用用户等。

错误响应统一为：

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request",
    "details": {}
  }
}
```

UI 应优先展示 `message`，必要时展开 `details` 给管理员或系统管理员。

## 异步任务与实时进度

生图任务通过 `/api/generate` 创建，随后通过 WebSocket 接收进度和图片事件。当前前端状态参考 [`web/src/stores/session.ts`](../web/src/stores/session.ts) 与 [`web/src/composables/useTaskWebSocket.ts`](../web/src/composables/useTaskWebSocket.ts)。

UI 必须表达以下任务状态：

- `queued`：已排队。
- `running`：生成中，显示百分比进度。
- `succeeded`：生成成功，显示图片结果。
- `failed`：生成失败，显示错误并提供重试。

设计上需要避免用户误以为页面卡住。任务可能耗时较长，刷新页面后仍应能恢复当前进行中的任务或从历史查看结果。

## 配额与计费感知

本产品按生成图片张数预扣配额。前端需要在以下位置体现额度：

- 全局侧栏用户卡片：剩余/总额度或无限。
- 工作台顶部：剩余额度。
- 生成提交前：当用户选择张数时，应让用户理解本次会消耗多少额度。
- 管理员用户详情：额度使用率、剩余额度、配额流水。
- 密钥管理：服务商密钥的已用/总额度。

普通用户额度不足时，提交入口应禁用或在提交失败后给出明确解释。非服务商错误导致的任务失败可能由后端退还额度，前端只展示最终权威额度。

## 安全与隐私约束

- API Key 明文只在创建或替换时输入一次，列表和详情只显示 hint。
- 不要在任何界面、日志或错误详情中暴露明文上游密钥。
- 图片资源来自私有 R2，通过鉴权接口访问。
- 普通用户只能看到自己的会话和图片。
- 系统管理员会话审计是只读运维能力，应在 UI 上与普通历史明确区分。
- 所有非 GET 请求依赖 CSRF Cookie 与 `X-CSRF-Token`，前端请求层已处理，重构时不要绕过 [`web/src/api/client.ts`](../web/src/api/client.ts) 的行为。

## 数据与接口参考

| 功能           | 主要接口                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------ |
| 登录与当前用户 | `POST /api/auth/login`、`POST /api/auth/logout`、`GET/PATCH /api/me`                             |
| 生成任务       | `POST /api/generate`、`GET /api/tasks/:id`、`POST /api/tasks/:id/retry`、`GET /api/ws/task/:id`  |
| 会话           | `GET/POST /api/sessions`、`GET/PATCH/DELETE /api/sessions/:id`、`GET /api/sessions/:id/messages` |
| 历史           | `GET /api/history`、`GET /api/history/:id`                                                       |
| 图片与上传     | `GET /api/i/:imageId`、`POST /api/uploads`                                                       |
| Prompt 助手    | `POST /api/prompt-assistant/turn`                                                                |
| 案例库         | `GET /api/prompt-cases`、`/api/sysadmin/prompt-cases*`                                           |
| 公告           | `/api/announcements*`、`/api/sysadmin/announcements*`                                            |
| 管理员用户     | `/api/admin/users*`、配额、用量、重置密码                                                        |
| 系统管理       | `/api/sysadmin/provider-keys*`、dashboard、preferences、settings、session inspection             |

完整接口以 [`API.md`](./API.md) 和本地 OpenAPI `http://localhost:8787/api/openapi.json` 为准。

## 重设计交付建议

建议外部协作者至少交付以下内容：

- 信息架构图：角色、导航、页面关系、默认入口。
- 核心流程线框图：登录、AI 图像生成、工作台、历史详情、用户管理、密钥管理、案例管理、会话审计。
- 关键页面高保真稿：桌面端和移动端普通用户创作流程；桌面端后台管理流程。
- 组件状态稿：按钮、输入、选择器、上传、表格、分页、弹层、Toast、空状态、失败状态、加载状态、全屏图片查看器。
- 设计 Tokens：颜色、字体、间距、圆角、阴影、图表色板、明暗主题。
- 交互说明：任务运行锁定、参考图上传、Prompt 助手、多图查看、危险操作确认。
- 可实现性说明：哪些交互依赖现有 API，哪些需要新增 API。

## 验收清单

重设计稿进入开发前，应逐项确认：

- 普通用户能在 3 步内从首页开始一次文生图或图生图。
- 用户能清楚区分案例 Prompt、助手生成 Prompt 和自己手写 Prompt。
- 当前服务商不支持的模式不会误导用户点击。
- 任务运行中有明确进度、锁定和失败恢复路径。
- 历史里能复查图片、Prompt、参数、参考图和失败原因。
- 管理员能在一页内完成搜索用户、加配额、重置密码和禁用用户。
- 系统管理员能安全创建和测试密钥，且明文密钥不会被二次展示。
- 案例库能支撑用户端内容运营，包括筛选、批量发布、预览和导入。
- 公告能完成草稿、预览、发布、撤回、归档和删除。
- 会话审计能支持排障，能看到失败图片、耗时、模型和 Prompt。
- 中文和英文文案都能容纳，不出现按钮或表格文字溢出。
- 浅色、深色、移动端、桌面端都至少有主要状态覆盖。
